vue-cli项目攻略

创建项目

  • 在Powershell或者VScode根文件夹下右键—在终端中打开, 输入vue create XXX(文件名)
  • 选择需要安装的插件
    • vue-router 项目路由 ,可后续安装
    • vuex 项目存储库 , 可后续安装
    • node-sass scss
    • babel 基础

引入插件

  • 在Powershell或者VScode根文件夹下右键—在终端中打开, 输入cnpm/npm install XXX(插件) --save : : : install(缩写)i : : : save(缩写)s : : : cnpm 淘宝的镜像,国内网络,比npm安装速度要快点
  • 检查方式, 在项目里的package.json文件夹里看有没有引入的插件

插件通用的CSS样式修改问题

  • 把style里面的scoped属性去除即可.
  • 理由 : scoped是让其他页面无法使用当前页面的CSS样式,大部分插件库都引用的自己的CSS.

常用的插件库

  • elementUI(webUI)

    • 安装 : npm i element-ui -S

    • 完整引入 : 在main.js中引入

      import Vue from 'vue';
import App from './App.vue';
//以下内容是需要添加的
import ElementUI from 'element-ui';
import 'element-ui/lib/theme-chalk/index.css';

Vue.use(ElementUI);

new Vue({
el: '#app', //还有这里也是需要添加的 
render: h => h(App)
});


  • vant(移动端UI)

    • 安装 : npm i vant -S

    • 完整引入 : 在main.js中引入

      import Vue from 'vue';
import App from './App.vue';
//以下内容是需要添加的
import Vant from 'vant';
import 'vant/lib/index.css';

Vue.use(Vant);

    • vant的px尺寸转rem

      1. 安装flexible和 postcss-px2rem (这两个CNPM找不到,只能用npm)
        npm install lib-flexible --save 和npm install postcss-px2rem --save
      2. 在项目入口文件main.js 中引入lib-flexible
        import 'lib-flexible'
      3. 在vue-cli3 项目中新建vue.config.js文件,有的话就直接用
         module.exports = {
     css: {
         loaderOptions: {
         css: {},
         postcss: {
             plugins: [
             require('postcss-px2rem')({
                 remUnit: 37.5
             })
             ]
         }
         }
     },
 }
      4. npm run serve 重启,必做


  • mockjs(随机生成) : npm install mockjs
    在需要数据的JS页面引入

      //这里只是用了mock的随机数据功能,其他功能参考这种引入
  import mocks from "mockjs";
  var mock = mocks.Random;


router路由

手动安装router的引入

  1. 安装router cnpm install vue-router --save

  2. 在src目录下新建router => router.js

  3. 在router.js里面添加import和export

     import Vue from "vue";
 import Router from "vue-router"
 Vue.use(Router)
 const router = new Router({
     mode:"history", // 需要后端配置一下
     scrollBehavior (to, from, savedPosition) {
         return new Promise((resolve, reject) => {
             setTimeout(() => {
             resolve({ x: 0, y: 0 })
             })
         })
     },
     routes:[
         {
             path:"/",
             redirect : "/login"
         },
         {
             path:"/login",
             component: () => import( '@/views/login/index.vue'),
             meta:{
                 isFootShow:false
             }
         },
     ],

 })

 export default router

  4. 在全局main.js中引入,地址根据实际情况修改

     import router from "./router/router.js"


转换页面后自动显示页面顶端(scrolltop为0)

在router.js里的router对象里

scrollBehavior (to, from, savedPosition) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
        resolve({ x: 0, y: 0 })
        })
    })
},


重定向redirect

在路径配置里面,使用重定向就不用import页面路径了

{
    path:"/",
    redirect : "/login"
},


页面导航router-link的显示和隐藏 meta属性

  1. 在router.js里面每个路径配置里添加meta属性
     routes:[
 {
     path:"/login",
     component: () => import( '@/views/login/index.vue'),
     meta:{
         isShow:false
     }
 },
 ],
  2. 在App.vue页面 id=”app”的导航链接标签里添加v-if判断$route.meta.名称
     <div id="nav" v-if="$route.meta.isshow">
     <router-link to="/">Home</router-link>&nbsp;&nbsp;|&nbsp;&nbsp;
     <router-link to="/user">User</router-link>
 </div>


Vuex

手动安装vuex并引入

  1. 安装vuex cnpm install vuex --save

  2. 在src下新建store => store.js

  3. 在store.js中添加import和export

     import Vue from "vue"
 import Vuex from "vuex"
 Vue.use(Vuex);

 const store = new Vuex.store({

 })

 export default store

  4. 在main.js中引入,地址根据实际情况修改

     import store from "./store/store.js"

vuex的state属性

相当于data,用于存放数据
获取方式 :this.$store.state.获取的值
模块化 :this.$store.state.模块名.获取的值

vuex的mutations属性

相当于method,处理同步的方法
获取方式 :this.$store.commit("函数名称")
模块化 :this.$store.commit.("模块名/函数名称")

vuex的action属性

相当于method,处理异步的方法
获取方式 :this.$store.dispatch("函数名称")
模块化 :this.$store.dispatch.("模块名/函数名称")

vuex的getters属性

相当于computed属性
获取方式 :this.$store.getters.获取的值
模块化 :this.$store.getters[模块名/函数名称]

在其他页面的引用方式

  • state和getters 在computed里面接收,
  • mutations和action 在method里面接收
    //这里的state(){} , getters(){} , mutations(){} ,action(){}
//对应的是当前页面使用的数据/方法的名称
computed: {
  state(){
      return this.$store.state.data
  },
  getters(){
      return this.$store.getters.data
  }
},
methods: {
  mutations(){ //可传值
    this.$store.commit("uptCount",val)
  },
  action(){ // 可传值
    this.$store.dispatch('uptCount', val)
  }
}

辅助函数引用

在页面中先引用 :
import { mapState, mapMutations, mapActions, mapGetters } from "vuex";

import { mapState, mapMutations, mapActions, mapGetters } from "vuex";
export default {
    computed: {
        ...mapState({
        state: function(state) {
            return state.data;
                }
            }),
        ...mapGetters({
            getters:"getters"
            })
        }),
    methods: {
        ...mapMutations({
        add: "add"
        }),
        ...mapActions({
        add: "add"
        })
    },
  },
}

vuex的module属性

module : 模块化,应用于大项目,数据较多的情况,把一个store.js内的数据分成很多份。

  1. 在store文件夹下新建modules文件夹, modules 文件夹存放的是不同的js模块 ,这里新建一个user.js

  2. 现在目录结构: store文件夹=>store.js和modules文件夹==modules文件夹里面==> user.js

  3. 在store.js中引入模块化的user.js

     import Vue from "vue"
 import Vuex from "vuex"
 Vue.use(Vuex);

 import user from "./modules/user"
 import car from "./modules/car"
 const store = new Vuex.Store({
     modules: {
     //其他文件使用的名称 : export导出的名称
         user: user,
         car:car
     }
 })

 export default store

  4. user.js文件中引入和导出

     const user = {
     state: {},
     //模块化的,必须写namespaced:true 不然下面三个不能使用
     namespaced: true,

     mutations: {},
     actions: {},
     getters: {}
 }
 export default user
